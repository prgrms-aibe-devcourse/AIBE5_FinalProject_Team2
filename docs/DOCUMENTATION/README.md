**대상**: Backend 개발자, Frontend 개발자, API 사용자

**활용 예시**:

```
Q: "사용자 프로필을 조회하려면?"
A: 문서에서 "1. 사용자 관리 > 1.2 프로필 조회" 참조

**목적**: "우리가 뭘 만드는가?"를 명확히 합니다

**대상**: 제품 관리자, 백엔드 리드, 팀 리드

**주요 내용**:

- ✅ 기능 요구사항 (FR-1 ~ FR-9)
- ⚙️ 비기능 요구사항 (성능, 보안, 신뢰성)
- 📊 성공 메트릭

**읽어야 할 섹션**:

- 제품 기획자: "개요" + "기능 요구사항" + "성공 메트릭"
- 개발 리드: "기능 요구사항" + "비기능 요구사항"
- QA: "기능 요구사항" + "성공 메트릭"

**활용 예시**:

```
Q: "신뢰도 검증(Trust Score)는 어떻게 구성되는가?"
A: 문서에서 "FR-4.3 Trust Score 계산" 참조
   - 백테스트 신뢰도 (30%)
   - 강건성 (25%)
   - 일관성 (25%)
   - 설명가능성 (20%)

### 5️⃣ [성능 최적화 가이드](05_PERFORMANCE_OPTIMIZATION.md)

**목적**: 느린 부분을 찾고 빠르게 만듭니다

**대상**: 백엔드 엔지니어, DevOps, 인프라 팀

**주요 내용**:

- 📊 성능 측정 & 목표
- ⚡ 백엔드 최적화 (캐싱, N+1, 비동기)
- 🗄️ 데이터베이스 최적화 (인덱싱, 쿼리)
- 🎨 프론트엔드 최적화 (번들 크기, 이미지)
- 🔬 분석 엔진 최적화 (벡터화, 병렬화)
- 📈 인프라 확장 (로드밸런싱, 레플리카)

**읽어야 할 섹션**:

- 백엔드 개발자: "백엔드 최적화" + "데이터베이스 최적화"
- 프론트엔드 개발자: "프론트엔드 최적화"
- DevOps/인프라: "인프라 확장" + "모니터링"

**활용 예시**:

```
Q: "API 응답이 450ms에서 200ms로 줄이려면?"
A: 문서에서 "백엔드 최적화" 참조
   1단계: N+1 제거 (450 → 250ms)
   2단계: 캐싱 추가 (250 → 150ms)
   3단계: 인덱싱 (150 → 100ms)
```
