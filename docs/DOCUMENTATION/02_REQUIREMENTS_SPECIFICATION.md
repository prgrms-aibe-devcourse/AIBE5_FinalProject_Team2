## 프로젝트 개요

### 비전

**"자유로운 삶을 위한 나만의 AI 퀀트 매니저"**

투자자의 **삶의 목표**를 **투자 전략**으로 변환하고, **신뢰할 수 있는 검증**을 통해 **스스로 판단하도록 돕는** AI 기반 투자 의사결정 플랫폼입니다.

### 핵심 가치 (4V)

| 가치               | 정의                                        | 구현                              |
| ------------------ | ------------------------------------------- | --------------------------------- |
| **Freedom** 자유   | 투자의 목적은 차트가 아니라 삶의 자유       | 개인의 재정 목표 중심 전략 설계   |
| **Clarity** 명확성 | 복잡한 금융을 이해 가능한 전략으로 변환     | AI 설명(SHAP), 규칙 명문화        |
| **Trust** 신뢰     | 단순 수익률이 아니라 왜 믿을 수 있는지 검증 | 강건성 테스트, 몬테카를로 분석    |
| **Agency** 주체성  | 사용자가 AI에게 맡기는 게 아니라 함께 판단  | 협력적 인터페이스 & 의사결정 로그 |

## 핵심 목표

### 비즈니스 목표

1. **신뢰의 플랫폼 구축**: 투자자가 **신뢰하는** 의사결정 도구 1위 달성
2. **사용자 기반 확장**: 초년도 월간활동사용자(MAU) 10,000명 달성
3. **수익화**: 구독료(Pro/Premium) 및 API 라이선스 기반 수익화
4. **고객 만족도**: NPS 70 이상 달성
5. **AI 신뢰도**: Trust Score 평균 85점 이상 유지

### 제품 목표

1. **자동 전략 생성**: 자연어 입력 → 백테스트된 전략 완성까지 < 5분
2. **검증의 완전성**: 모든 전략에 대한 강건성 검증 및 신뢰 점수 제공
3. **설명의 명확성**: 투자자 누구나 전략을 이해할 수 있는 수준의 설명 제공
4. **의사결정 추적**: 사용자의 모든 투자 의사결정 로그 축적 및 학습

## 사용자 구분

### 투자자 (Investor)

**정의**: 자신의 재정 목표를 투자로 달성하려는 개인

**핵심 역할**:

- 재정 목표 및 투자 관심사 정의
- AI 제안 검토 및 전략 선택
- 포트폴리오 관리 및 모니터링
- 의사결정 기록 및 반복 개선

**요구사항**:

- ✅ 이메일 인증
- ✅ 투자 목표 설정
- ✅ 위험도 평가 (CONSERVATIVE, MODERATE, AGGRESSIVE)
- ✅ 초기 자본 등록

### 전문가 / 트레이더 (Trader/Quant)

**정의**: 수익 창출을 목표로 활발하게 거래하는 고급 사용자

**추가 역할**:

- 커스텀 전략 개발
- API를 통한 자동매매 연동
- 신호 패키지 구독

**요구사항**:

- ✅ Investor 모든 조건
- ✅ API 키 발급 가능
- ✅ 강고급 분석 도구 접근권

### 기관 / 펀드 (Institutional)

**정의**: 복수의 포트폴리오를 관리하는 기관 투자자

**추가 역할**:

- 팀 협업 (여러 매니저)
- 대규모 백테스트
- 맞춤형 리포팅

## 기능 요구사항 (FR)

### FR-1: 사용자 인증 및 온보딩

#### FR-1.1 회원가입

**설명**: 새로운 투자자가 계정을 생성합니다.

**입력**:

- 이메일 (유일)
- 비밀번호 (8자 이상, 숫자+특수문자)
- 사용자명 (2~20자)

**처리**:

- 이메일 중복 확인
- 비밀번호 강도 검증
- JWT 토큰 발급

**출력**:

- 사용자 ID
- JWT 토큰
- 온보딩 상태 (`NEEDS_PROFILE`)

**검증**:

- [x] 이메일 유효성 (RFC 5322)
- [x] 비밀번호 강도
- [x] 약관 동의

#### FR-1.2 재정 목표 설정

**설명**: 사용자가 투자 목표를 정의합니다.

**입력**:

- `investmentGoal`: "10년 안에 3억 모아서 자유로운 생활"
- `timeHorizon`: 1Y, 3Y, 5Y, 10Y
- `targetAmount`: 300000000
- `riskTolerance`: CONSERVATIVE / MODERATE / AGGRESSIVE
- `initialCapital`: 5000000

**처리**:

- 목표의 현실성 검증 (CAGR 필요%)
- AI가 적절한 자산배분 제안

**출력**:

- 목표 저장됨
- AI의 자산배분 제안 (주식%, 채권%, 현금%)

#### FR-1.3 프로필 완성

**설명**: 사용자의 투자 프로필을 완성합니다.

**입력**:

- 투자 경험 (초보/중급/전문가)
- 월 여유자금
- 거래 빈도
- 관심 자산군 (주식, 채권, 암호화폐 등)

**출력**:

- 프로필 저장
- 온보딩 완료 (`ACTIVE`)

### FR-2: 전략 생성 (AI 기반)

#### FR-2.1 자연어 전략 설명

**설명**: 사용자가 원하는 투자 전략을 자연어로 설명하고, AI가 이를 구조화합니다.

**입력**:

```
"배당금으로 생활비를 충당하고 싶어.
연 4% 정도의 배당수익률을 목표로 하되,
주가 하락 시 손절은 하고 싶지 않아.
변동성은 낮춰야 해."
```

**처리 흐름**:

1. LLM(Google Gemini)이 목표 분석
2. AI가 전략 규칙 생성:
   - 자산 선정: 고배당주 (배당률 > 3.5%)
   - 진입 신호: 배당락 전, 저PER 상태
   - 위치 크기: 균등 배분
   - 재조정: 분기별
3. 규칙 이해도 검증 (사용자가 설명을 이해하는지)

**출력**:

- 전략 ID: `strategy_20260524_001`
- 전략명: "배당 성장 포트폴리오"
- 규칙:
  ```json
  {
    "assetSelection": "High Dividend Stocks (Div Yield > 3.5%)",
    "entrySignal": "Ex-Dividend Date 이전 + RSI < 40",
    "positionSize": "균등 배분 (N = 10종목)",
    "rebalance": "분기별",
    "exitSignal": "배당락 후 10% 이상 하락"
  }
  ```

#### FR-2.2 템플릿 기반 전략 생성

**설명**: 사용자가 사전정의된 템플릿을 선택합니다.

**템플릿 목록**:

- `DIVIDEND_GROWTH`: 배당 성장 포트폴리오
- `MOMENTUM`: 모멘텀 추종 전략
- `VALUE_INVESTING`: 가치 투자
- `QUALITY_FACTOR`: 품질 지수 투자
- `BALANCED`: 균형 포트폴리오
- `GROWTH`: 성장주 중심
- `DEFENSIVE`: 방어주 중심

**처리**:

- 사용자의 프로필과 목표에 맞춰 템플릿 커스터마이징
- 역사적 성과 데이터 제공

**출력**:

- 커스터마이즈된 전략 & 규칙
- 선택한 자산군

### FR-3: 백테스트 및 성능 분석

#### FR-3.1 백테스트 실행

**설명**: 전략의 과거 성과를 검증합니다.

**입력**:

- `strategyId`: 101
- `startDate`: "2020-01-01"
- `endDate`: "2026-05-24" (현재)
- `initialCapital`: 10000000
- `slippage`: 0.1% (거래 마찰)
- `commission`: 0.1% (수수료)

**처리**:

1. Python VectorBT 엔진으로 시뮬레이션
2. 매매 시그널 생성
3. 포트폴리오 가치 추적
4. 성과 지표 계산

**출력** (성과 지표):

```json
{
  "totalReturn": 152.3,           // %
  "annualizedReturn": 14.5,       // % p.a.
  "maxDrawdown": -18.2,           // %
  "sharpeRatio": 1.87,            // 위험조정 수익률
  "sortinoRatio": 2.45,           // 하방 변동성 기준
  "winRate": 58.5,                // 수익 거래 비율
  "profitFactor": 2.15,           // 수익/손실 비율
  "trades": 142,                  // 총 거래 수
  "avgTradeDuration": 23,         // 일 단위
  "equityCurve": [...]            // 시계열 자산가
}
```

#### FR-3.2 드로우다운 분석

**설명**: 포트폴리오의 손실 구간을 분석합니다.

**출력**:

```json
{
  "maxDrawdown": -18.2,
  "maxDrawdownDate": "2020-03-16",
  "recoveryDays": 187,
  "drawdowns": [
    {
      "start": "2020-03-01",
      "end": "2020-03-16",
      "maxDD": -18.2,
      "recovery": "2020-09-21",
      "reason": "COVID-19 팬데믹"
    }
  ]
}
```

### FR-4: 신뢰도 검증 (Robustness & Trust Score)

#### FR-4.1 몬테카를로 시뮬레이션

**설명**: 백테스트 결과의 견고성을 검증합니다.

**처리**:

1. 수익률 분포 추정
2. 1,000회 시뮬레이션 수행
3. 반환 분포 분석

**입력**:

- 백테스트 결과
- 시뮬레이션 횟수 (기본: 1,000회)
- 신뢰 수준 (기본: 95%)

**출력**:

```json
{
  "returnDistribution": {
    "mean": 145.2,
    "stdDev": 32.5,
    "percentile5": 85.3, // 최악의 5%
    "percentile25": 120.5,
    "percentile50": 145.2,
    "percentile75": 169.8,
    "percentile95": 205.1 // 최고의 5%
  },
  "drawdownDistribution": {
    "mean": -15.8,
    "worst": -35.6
  },
  "probability": {
    "doubleCapital": 0.87, // 자본 2배 확률
    "positiveReturn": 0.94 // 양수 수익 확률
  }
}
```

#### FR-4.2 과적합 탐지

**설명**: 백테스트가 과거에만 잘 맞는지 검증합니다.

**방법**:

- Walk-Forward Analysis
- Out-of-Sample 테스트
- Parameter Sensitivity 분석

**출력**:

```json
{
  "overfittingRisk": "LOW", // LOW / MEDIUM / HIGH
  "confidence": 0.92, // 신뢰도
  "message": "다양한 시기와 조건에서 일관된 성과 검증됨"
}
```

#### FR-4.3 Trust Score 계산

**설명**: 전략의 종합 신뢰도를 계산합니다.

**구성 요소**:

1. **백테스트 신뢰도** (30%)
   - 충분한 거래 수 (최소 30회)
   - 충분한 데이터 기간 (최소 2년)
   - 낮은 과적합 위험
2. **강건성** (25%)
   - 몬테카를로 통과 확률
   - 스트레스 테스트 결과
3. **일관성** (25%)
   - 연도별 성과 일관성
   - 시장 사이클별 안정성
4. **설명가능성** (20%)
   - 규칙의 명확성
   - SHAP 값의 해석 용이성

**출력**:

```json
{
  "trustScore": 87.5, // 0 ~ 100
  "components": {
    "backtestReliability": 90,
    "robustness": 82.5,
    "consistency": 88.3,
    "explanability": 85.6
  },
  "rating": "HIGHLY_TRUSTWORTHY", // LOW / MEDIUM / HIGH
  "lastUpdated": "2026-05-24T10:30:00Z"
}
```

### FR-5: 설명 및 인사이트

#### FR-5.1 전략 설명 (Strategy Explanation)

**설명**: 투자자가 전략을 이해할 수 있도록 설명합니다.

**SHAP 기반 설명**:

```
이 전략의 핵심:

1. 진입 신호 (Importance: 60%)
   - RSI < 30: 과매도 상태에서 반등 기회
   - 이동평균 크로스: 추세 전환 신호

2. 위치 관리 (Importance: 25%)
   - 최대 10%: 단일 주식 위험 제한
   - 분산: 최소 8개 종목 보유

3. 익절 규칙 (Importance: 15%)
   - 배당락 전 일부 수익실현 (전략의 본질: 배당 수익)
```

#### FR-5.2 거래별 설명 (Trade Explanation)

**설명**: 특정 거래가 왜 발생했는지 설명합니다.

**예시**:

```
거래 #42: AAPL 매수 (2023-06-15)

진입 신호:
- RSI = 28 (과매도) ✓
- 20일 MA가 50일 MA 위로 ✓

컨텍스트:
- VIX = 16 (낮은 변동성) → 진입 타이밍 양호
- 업계 뉴스: FDA 승인 긍정 뉴스
- 기술적: 지지선 상향 돌파

결과:
- 진입: $155.30 → 익절: $162.45 (+4.59%)
- 예상대로: 자동 배당락 매도로 종료
```

### FR-6: 의사결정 지원

#### FR-6.1 전략 실행 조언 (Advisor)

**설명**: AI가 전략 실행 여부를 조언합니다.

**AI 평가 기준**:

- ✓ Trust Score > 80?
- ✓ 강건성 테스트 통과?
- ✓ 사용자 위험도와 매치?
- ✓ 충분한 유동성?

**출력**:

```json
{
  "recommendation": "IMPLEMENT", // IMPLEMENT / HOLD / REVISE
  "confidence": 0.87,
  "rationale": "Trust Score 87점 이상, 강건성 검증 완료, 위험도 일치",
  "conditions": [
    "초기 포지션 작게 시작 (50% → 100% 증액)",
    "월 1회 리밸런싱 권장"
  ],
  "risks": [
    "고금리 환경에서 배당주 다소 약세 가능",
    "시장 급락 시 배당금 삭감 리스크"
  ]
}
```

#### FR-6.2 의사결정 로그

**설명**: 사용자의 모든 투자 결정을 기록합니다.

**기록 항목**:

- `decision`: IMPLEMENT / SKIP / MODIFY
- `timestamp`: 의사결정 시점
- `strategy`: 관련 전략
- `aiRecommendation`: AI의 조언
- `userNotes`: 사용자의 판단 이유
- `userRating`: 결정에 대한 만족도 (1~10)

**활용**:

- 사용자의 투자 성향 학습
- 의사결정 성과 추적
- AI 모델 개선을 위한 피드백

### FR-8: 리포팅 및 분석

#### FR-8.1 월간 리포트

**설명**: 월간 투자 성과를 리포트합니다.

**포함 내용**:
